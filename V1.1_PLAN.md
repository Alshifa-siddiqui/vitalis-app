# Vitalis — v1.1 Plan (post-launch native features)

These two need native modules + an EAS rebuild, so they're intentionally saved
for after the v1.0 launch. Each is scoped so it can be picked up cleanly.

---

## 1. Home-screen widget (Android)

**Why:** the single most-requested retention feature for habit apps — a
glanceable daily nudge + one-tap logging without opening the app.

**Library:** `react-native-android-widget` (Expo config plugin; lets you define
widgets with RN-style components and a background task handler).

**Phasing:**
- **v1.1a — read-only widget:** shows today's date, "X of Y done", and the
  wellness ring. Tapping opens the app. Simplest; no cross-process writes.
- **v1.1b — interactive widget:** tap a habit to check it in from the home
  screen (updates via the widget task handler → writes to storage → app reads on
  next open).

**Technical steps:**
1. `npx expo install react-native-android-widget` + add its config plugin to
   `app.json`.
2. Build a `HabitWidget` layout component (widget-flavored primitives).
3. Register a `widgetTaskHandler` that reads today's habits/progress from a shared
   store (persist a small JSON snapshot to `AsyncStorage`/SharedPreferences on
   every check-in so the widget can read it out-of-process).
4. Update the widget after each in-app check-in (`requestWidgetUpdate`).
5. `eas build` (native rebuild required), test on device, add a widget screenshot
   to the store listing (widgets sell).

**Effort:** ~1–2 focused days. **Risk:** medium (out-of-process data sync).

## 2. Google Fit / Health Connect integration (Android)

**Why:** auto-complete health habits from real activity (steps, workouts) — less
friction, and it lets Vitalis sit *on top of* dedicated gym apps instead of
competing with them.

**Important:** Google Fit APIs are being retired in favor of **Health Connect**
(the current Android standard). Target Health Connect, not the legacy Fit API.

**Library:** `react-native-health-connect`.

**Scope (v1.1):**
- Read **Steps** and **ExerciseSession** (workouts) for today.
- Auto-check a habit when matching activity exists — e.g. a "Workout" habit
  auto-completes if Health Connect logged an exercise session today; a "10k
  steps" habit completes when steps ≥ goal.
- Surface a small "Synced from Health Connect ✓" note so it's transparent.

**Technical steps:**
1. `npx expo install react-native-health-connect` + config plugin.
2. Declare the Health Connect permissions (Steps, ExerciseSession) in config.
3. On app open (and pull-to-refresh): check availability → request permissions →
   read today's records → auto-complete linked habits.
4. Add a per-habit "link to Health Connect" option (which activity maps to it).
5. **Privacy:** update `PRIVACY.md` + the Play **Data Safety** form (declare
   health/fitness read access) — required.
6. `eas build`, test on a device with Health Connect installed.

**Effort:** ~2 days. **Risk:** medium (permissions UX + device variance).

## Sequencing
Ship **v1.0** first. Then v1.1a (read-only widget) → v1.1b (interactive) →
Health Connect. Each is its own EAS build; batch them if possible to save builds.
